## 1. 查看状态

每次操作前，建议先查看仓库状态：

```bash
git status
```

**作用：** 显示哪些文件被修改了、哪些文件还未被跟踪。

---

## 2. 添加文件

### 2.1 创建新文件

你可以用任何编辑器创建文件，或者用命令行：

```bash
echo "这是我的第一个练习文件" > practice.txt
```

### 2.2 添加到暂存区

```bash
# 添加指定文件
git add practice.txt

# 或者添加所有修改的文件
git add .
```

---

## 3. 提交修改

```bash
git commit -m "这里写你的提交说明"
```

**提示：** 提交信息应该清晰地描述这次修改做了什么。

---

## 4. 推送到远程

```bash
git push
```

**说明：** 第一次推送后，之后只需要 `git push` 即可，不需要再写 `-u origin main`。

---

## 5. 查看历史

```bash
# 查看详细历史
git log

# 查看简洁历史
git log --oneline
```

---

## 6. 分支操作

### 6.1 创建新分支

```bash
git branch test-branch
```

### 6.2 切换分支

```bash
git checkout test-branch
```

### 6.3 创建并切换到新分支（常用）

```bash
git checkout -b new-feature
```

### 6.4 查看分支

```bash
git branch
```

### 6.5 合并分支

```bash
# 先切回主分支
git checkout main

# 合并目标分支
git merge new-feature
```

---

## 7. 完整工作流程

每次修改代码的标准流程：

```
1. 修改/创建文件
2. git status          （查看状态）
3. git add .           （添加到暂存区）
4. git commit -m "说明" （提交）
5. git pull            （先拉取最新代码）
6. git push            （推送到远程）
```
