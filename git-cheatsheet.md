# 🚀 Git 极速查表 (Cheat Sheet)

这份文档总结了 Git 最常用的核心命令，适合日常开发查阅。

## 💻 基础工作流

| 命令 | 说明 |
|------|------|
| `git status` | 查看当前文件修改状态 |
| `git add <文件名>` | 添加指定文件到暂存区 |
| `git add .` | 添加所有修改的文件到暂存区 |
| `git commit -m "说明"` | 提交修改到本地仓库 |
| `git push` | 将本地提交推送到远程 GitHub |
| `git pull` | 拉取远程最新代码并合并 |

---

## 🌿 分支管理 (Branches)

| 命令 | 说明 |
|------|------|
| `git branch` | 查看所有本地分支（带 `*` 的是当前分支） |
| `git branch <分支名>` | 创建一个新分支（不切换） |
| `git checkout -b <分支名>` | **创建并切换**到新分支（最常用） |
| `git checkout <分支名>` | 切换到已存在的分支 |
| `git merge <分支名>` | 将指定分支的代码合并到**当前所在分支** |
| `git branch -d <分支名>` | 删除指定分支 |

---

## ⏪ 撤销与恢复 (Undoing)

| 命令 | 说明 |
|------|------|
| `git checkout -- <文件名>` | 撤销该文件未 add 的修改 |
| `git reset HEAD <文件名>` | 将文件从暂存区撤回（还在修改状态） |
| `git reset HEAD~1` | 撤回最近一次 commit（修改保留在文件中） |
| `git revert HEAD` | 创建一个新的提交来抵消最近一次提交（安全撤销） |
| `git stash` | **保存现场**：暂时储藏未提交的修改，让工作区变干净 |
| `git stash list` | 查看所有储藏的记录 |
| `git stash pop` | **恢复现场**：取出最近一次储藏的修改并应用 |

---

## 🔍 查看差异与历史

| 命令 | 说明 |
|------|------|
| `git log` | 查看详细的提交历史 |
| `git log --oneline` | 查看简洁的一行式提交历史 |
| `git diff` | 查看文件具体修改了哪些内容（Diff 对比） |

---

## 🏷️ 标签与协作 (Tags & Collaboration)

| 命令 | 说明 |
|------|------|
| `git clone <URL>` | 从远程仓库克隆完整项目到本地 |
| `git remote -v` | 查看远程仓库的地址连接情况 |
| `git tag -a v1.0 -m "说明"` | 创建发布版本标签 (如 v1.0) |
| `git push origin <标签名>` | 推送标签到远程仓库 |

---

> 💡 **Tip**: 遇到不会的命令，随时可以使用 `git help <命令>` 查看官方文档！
