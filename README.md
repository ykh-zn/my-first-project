## 📂 项目简介

本项目旨在通过**真实的代码实践**，掌握 Git 的核心工作流。从最基础的仓库创建，到高级的分支管理、冲突解决以及团队协作，这里包含了开发者日常开发中 90% 以上会用到的 Git 技能。

---

## ✨ 核心亮点

- 🛠️ **完整工作流**：包含 `add`, `commit`, `push`, `pull` 的标准操作演示。
- 🌿 **分支管理**：实践了分支的创建、切换、合并以及 Pull Request (PR) 协作流程。
- 🛡️ **冲突解决**：模拟了多人协作时的代码冲突场景及标准处理方案。
- ⏪ **撤销与恢复**：掌握了 `reset`, `revert`, `stash` 等“后悔药”命令。
- 🏷️ **版本发布**：使用 Tag 成功标记了项目里程碑 (v1.0)。

## 🎓 学习路线 (Learning Path)

如果你也想从零开始学习，可以参考我走过的路：

1. **第一阶段：入门**
   - 安装并配置 Git 用户信息
   - 学会 `init`, `clone`, `status`
2. **第二阶段：日常操作**
   - 熟练掌握 `add`, `commit`, `push`, `pull`
   - 编写规范的 `.gitignore`
3. **第三阶段：分支与协作**
   - 掌握 `branch`, `checkout`, `merge`
   - 在 GitHub 上发起 Pull Request
   - 解决合并冲突 (Merge Conflicts)

## 🛠️ 常用命令速查

```bash
# 提交代码三板斧
git add .
git commit -m "清晰的提交信息"
git push

# 查看状态
git status
git log --oneline
```

> 💡 更详细的命令请参考项目内的 **[git-cheatsheet.md](./git-cheatsheet.md)**
